# 课程试讲与迭代

课程迭代不是收集“喜欢不喜欢”，而是检查学员能否理解、执行并把方法迁移到真实场景。每轮只验证少量明确假设，让反馈能够改变下一步课程决定。

## 最小闭环

```text
课程版本 → 可被推翻的教学假设 → 最小试讲 → 真实行为与原话
        → 制作方解释 → 最小修订 → 下一轮结果回填
```

原始证据、学员表达和制作方解释必须分层。后续结果可以修正解释，但不能回写最初假设，让历史看起来从未犯错。

## 四类信号

- 理解：学员能否复述关键判断，并区分相近概念。
- 执行：学员能否独立完成课程要求的动作或产物。
- 迁移：情境略有变化时，学员能否判断该继续、调整还是停止。
- 摩擦：学员在哪一步停住、误解、绕开或必须求助。

完成率和满意度可以辅助判断，但不能单独证明学会。优先观察作业、操作记录、复述、真实提问和后续行为。

## 修改梯度

按证据指向选择最小变更：

1. 术语不清：补定义、反例或对照。
2. 步骤不会做：拆动作、补模板或示范。
3. 会做但不会判断：补边界、决策条件和失败案例。
4. 前置知识缺失：调整顺序或增加桥接练习。
5. 学完仍无法兑现成果：重新检查课纲与课程定位。

单次个人偏好不触发全课重构。事实错误、安全风险和明确不可执行步骤可以凭单条强证据立即修复，其余结构性改动需要重复信号或不同证据交叉支持。
