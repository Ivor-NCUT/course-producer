# 课程海报策略与原型

负责海报购买逻辑、文字方案和可编辑版式原型，不生成不可编辑的最终商业成片。开始前读取完整课程、定位、课纲、讲师/权益/价格等销售信息，以及 `知识库/方法论/课程海报.md` 和 `知识库/模板/课程海报方案模板.md`。

## 先拆材料

提取并保留来源：课程承诺、目标人群与迫切问题、讲师信任证据、课程大纲、学习成果、服务权益、案例反馈、价格、活动机制、购买须知和行动入口。

材料中没有的价格、名额、案例、背书、课时和赠品一律写 `[待补：字段]`，不得根据同类课程补造。

## 判断海报类型

选择一个主类型并说明证据：

- `conversion_training`：结果明确、周期短、陪跑/价格/限时机制强；
- `rational_professional`：内容复杂、客单较高，需要机制、课纲与证据逐步建立信任；
- `instructor_trust`：购买主要来自讲师个人品牌、履历和成果；
- `practical_outcome`：面向 IP、副业或实操，重点是路径、作业、成果样例与陪跑。

类型决定信息顺序和模块比例，不套统一模板。

## 输出文字方案

生成 `.course-producer/artifacts/poster-plan.md`，必须包括：

- 海报定位与主购买理由；
- 建议使用长图、4–5 张系列图或两者；
- 主标题、补充说明、主承诺与目标人群；
- 讲师/信任证据、成果、课程大纲、权益、案例、购买须知和 CTA；
- 每个模块的相对高度、信息密度、图片/截图/二维码位置和存在理由；
- 所有待补字段及它们对转化的影响。

主标题、讲师/信任、课程大纲、你将获得、适用人群和购买引导是必写模块。只复制课程目录不算海报方案。

## 可编辑画板原型

用户目标包含画板且当前 `lark-cli` 已授权时，调用 `beautiful-feishu-whiteboard`：

1. 运行其 preflight，读取 `RULES.md`、`CATALOG.md` 和所选模板 `design.md`；
2. 把文字方案编排为完整长图骨架或 4–5 个连续竖版画幅；
3. 只用可编辑 SVG 原生形状和 `<text>`，不设置 `font-family`，不把任务说明、来源说明和风格名写上画布；
4. 缺失材料用低饱和 `[待补]` 占位框；
5. 本地渲染后查看图片，修复文字溢出、重叠、边距、比例和二维码留白；
6. 写入飞书文档后再查看 live board，回读文档与画板并记录链接。

当前环境没有 `lark-cli`、用户授权或目标文档权限时，交付完整文字方案和 blocker，到此停止；不要声称画板已创建，也不要发起群聊 OAuth。

## 完成检查

- 海报类型有课程证据，信息顺序与类型一致。
- 主承诺、目标人群、信任证据、大纲、权益、CTA 和模块比例齐全。
- 所有数字、案例、背书、价格和优惠有来源或待补占位。
- 首屏能快速说明课程是什么、给谁、解决什么问题。
- 画板原型可编辑并已完成本地与 live 视觉检查，或明确停在文字方案边界。
