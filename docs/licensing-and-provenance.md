# 许可与来源边界

## 结论

Course Producer 是独立原创实现。它研究了 [dontbesilent2025/dbskill](https://github.com/dontbesilent2025/dbskill) 的产品结构，但不复制该仓库的受限内容。

dbskill 当前采用 CC BY-NC 4.0。其“主路由 + 多个细分 Skill + 文件型知识库”的公开结构可用于理解一种架构思路；其具体 `SKILL.md`、知识原子、方法论文档、脚本、图片和文案不进入 Course Producer。

## 来源分层

| 类别 | 可进入本仓库的内容 | 处理规则 |
|---|---|---|
| 用户指定的三个本地课程 Skill | 课程方法、流程、质量标准、命令模式 | 重写并按新边界拆分，保留来源记录 |
| 用户后续提供的课程业务材料 | 仅在用户明确要求沉淀时进入通用知识库 | 默认只用于当前课程项目 |
| dbskill | 架构研究结论 | 只记录“借鉴实现”决策，不复制内容 |
| 已安装伴随 Skills | 调用接口和兼容说明 | 不把伴随 Skill 文件打进本仓库 |
| 平台与 CLI 文档 | 必要命令和验证模式 | 只保留最小、可验证的操作说明 |

## dbskill 研究结论

截至 2026-07-20 研究时，上游公开仓库表现为：

- 一个统一入口负责读取上下文与动态路由；
- 多个 Skill 各自完成细分任务；
- `知识库/` 下使用 Markdown、JSONL 和词典；
- 构建工具把 Skills 与它们引用的知识文件打包；
- 用户状态保存在本地文件而不是远程服务。

本项目采用“借鉴局部”的结论：

- 采用主路由、原子 Skill、知识库分层和本地状态的思想；
- 为课程生产重新设计所有职责、协议、文件结构和文本；
- 不复制其路由表、知识字段内容、构建代码或发布文案；
- 不把 dbskill 作为运行时依赖。

## 后续发布规则

- 发布前重新核对本仓库全部文件的来源。
- 知识原子必须带 `source` 字段。
- 不能确认再分发权的外部材料，只保留引用或调用说明。
- 本地密钥、token、课程客户材料和真实项目状态不得提交。
- 根级许可证只能覆盖本仓库原创内容，不能改变外部工具或用户材料的原有权利。
