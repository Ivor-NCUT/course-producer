# 课程制作人：把一堆课程素材持续做成一门完整课程

## 先说结论

当你手里有访谈录音、飞书妙记、Word、PDF、零散课纲或一批半成品课程稿，不想每次重新向 Agent 解释进度，可以用“课程制作人”。它会先建立材料和项目状态，再沿着定位、访谈、知识萃取、课纲、逐字稿、审校和交付逐段推进；任务中断后，它会检查真实产物，从第一个没有完成的阶段继续。

## 这个 skill 是做什么的

课程制作人是一组相互配合的 Agent Skills。主入口判断现在该做哪一步，细分 Skill 各自只完成一件事：

- 材料盘点：登记每个文件和飞书来源，区分课程素材、执行指令、会议闲聊与不可读材料；
- 课程定位：明确课程给谁、解决什么挑战、学完能做什么，以及课程不承诺什么；
- 专家访谈：只围绕材料缺口设计主问题、追问条件和证据请求；
- 知识萃取：把观点、方法、案例、数据、边界和讲师原话变成可追溯知识卡；
- 课纲设计：让最着急的学员先拿到成果，后面的章节继续复用前面的概念和产物；
- 逐字稿：按章写可讲、可录、可审的正式课程稿；
- 课程审校：检查逻辑、事实、信息密度、口播节奏和 AI 写作坏味道；
- 飞书与海报：有授权时写入飞书、添加局部评论，并生成可编辑海报画板原型；
- 审美学习：从改稿和版本选择中提取候选偏好，经过多样本验证后才形成长期规则。

## 你在哪些工作场景能用到

- 你要从零做一门系统课，只有主题、讲师和少量访谈材料。
- 你已经访谈过行业专家，希望把几小时录音加工成课纲和逐字稿。
- 课程做了一半，素材、章节和评论散落在本地与飞书，想统一进度后继续。
- 你已经有完整课程稿，只想检查逻辑、事实与 AI 味，不希望推倒重来。
- 课程内容已经完成，需要飞书协作、审查评论或招生海报方案。

## 什么时候不要用它

- 只需要把一段录音逐字转写，不需要课程结构时，直接用妙记或音视频转写工具。
- 只改一句标题、一个自然段或一条海报文案时，直接用对应写作 Skill 更快。
- 需要课程平台开发、视频剪辑、商业投放或最终精修海报时，本 Skill 只能交付内容与可编辑原型，不能代替这些专业环节。
- 材料含未经授权的客户隐私、版权内容或商业秘密时，先处理权限与脱敏，再进入课程生产。

## 你需要输入什么

最低输入是一句话目标和素材位置。材料越清楚，课程制作人越少打断你。

建议提供：

- 课程项目所在的本地目录；
- 飞书文档、Wiki 或妙记链接；
- 已有课纲、逐字稿、讲师介绍和用户反馈；
- 目标学员、课程用途、预计成果和明确禁区；
- 本轮要完整生产、从某阶段开始，还是只做单项任务；
- 是否需要飞书写回、评论和海报。

可以直接复制这段输入：

```text
课程制作人，请读取 /path/to/course 里的全部课程素材。
课程给第一次带团队的业务负责人，目标是让他们能建立一套周会与复盘机制。
请先建立项目状态，再连续完成定位、知识萃取、课纲、逐字稿和审校。
没有证据的数字与案例保留待确认，不要编造；暂时不用写飞书和做海报。
```

## 它会输出什么

课程项目会出现一个 `.course-producer/` 目录，其中包含：

- `materials.jsonl`：所有材料的来源、指纹、状态、用途和证据位置；
- `decisions.jsonl`：会改变课程方向的待决策问题；
- `positioning.md`：课程定位；
- `interview-guide.md`：动态专家访谈提纲；
- `knowledge-cards.jsonl`：带来源的知识卡；
- `course-blueprint.md`：成果优先课纲；
- `lessons/*.md`：逐章课程稿；
- `review.jsonl`：审校问题、证据、动作和处理状态；
- `state.json`：阶段、产物、验证结果、阻塞项和下一步。

## 示范任务 1：从访谈和资料做成完整课程

### 你可以这样输入

```text
课程制作人，读取这个目录：/Projects/AI-sales-course。
里面有两条飞书妙记、一本 PDF、讲师的旧课纲和三篇公众号文章。
目标学员是第一次负责 AI 销售的创始人。请持续做成一门完整课程，先不写飞书；遇到会改变目标学员或课程承诺的问题再停下来问我。
```

### 示范运行后的输出

```text
已完成材料摄取：8 项可读，1 项缺权限，2 段会议闲聊已排除。
课程定位：帮助第一次负责 AI 销售的创始人，在现有团队与 30 天验证周期内，完成客户假设、访谈和第一轮销售动作。
下一阶段：已有访谈覆盖方法步骤，但缺少失败边界与一个完整成交案例；将生成补访谈提纲。
```

课纲节选：

```markdown
模块一：完成第一轮客户验证
面向：30 天内必须判断产品是否有人买的创始人
独立成果：一份客户清单、访谈问题卡和首轮触达记录
完成标准：至少能判断一个客户假设该继续、调整还是停止
```

### 这份输出怎么用

你可以先看定位和第一模块成果是否符合业务目标，再让 Agent 继续长时间生产。某个事实材料暂时缺失时，它会把内容留在待确认清单，不会因为缺一个数字就停止所有章节。

## 示范任务 2：精修已有课程，不推倒重来

### 你可以这样输入

```text
课程制作人，只审校 /Projects/course-v2/lessons 里的十章课程稿。
重点检查模块顺序、段落衔接、无来源数字和一行一句的字幕感。
保留讲师的判断与案例，不重新做定位和访谈；输出 review.jsonl 和修订稿，原稿不要覆盖。
```

### 示范运行后的输出

```json
{"review_id":"rev-017","artifact":"lessons/06.md","locator":"## 渠道筛选","category":"spoken_flow","severity":"medium","evidence":"连续五个单句段切断同一解释链","action":"auto_fix","status":"fixed"}
{"review_id":"rev-021","artifact":"lessons/08.md","locator":"paragraph:12","category":"unsupported_claim","severity":"high","evidence":"转化率 42% 没有来源","action":"needs_evidence","status":"open"}
```

修订稿会保留在独立目录。涉及承诺、品牌口径或讲师立场的问题不会自动改写，而会进入 `needs_decision`。

## 如果你觉得不满意，可以怎么提优化需求

可以直接复制这些反馈：

```text
这版课纲仍像知识目录。请让第一模块服务最着急的学员，并写出他们学完后能交付的具体成果。
```

```text
第 3 章太像整理过的访谈。请先明确教学任务，再用案例解释方法，最后给出学员能照着做的字段与标准。
```

```text
这版段落太碎。连续解释同一个动作时，请合并成自然句群，只在转场、提问和作业处单独成段。
```

```text
不要把讲师的判断改成通用结论。请恢复讲师署名，并把没有外部证据的数字标为待确认。
```

```text
我更喜欢 B 版，因为它先给判断，再展开具体场景。请记录为候选偏好，但先不要修改主 Skill，等另一个章节也验证后再决定。
```

```text
海报只保留有证据的讲师成果；价格、案例和二维码位置先用待补占位，不要参考同类课程补齐。
```

## 高阶用法

- 指定阶段：只调用课程定位、逐字稿、审校或海报原子 Skill。
- 断点恢复：说“课程制作人，继续上次项目”，它会核对状态与文件指纹。
- 按章检查点：让它先完成最重要的三章，确认后继续其余章节。
- 飞书协作：要求保留原稿创建同级正式稿，并只给需要 CEO 判断的内容添加局部评论。
- 审美对齐：给出原稿、你的改稿和选择理由，让候选偏好在不同章节回测后再沉淀。

## 使用时的判断标准

- 定位是否讲清楚给谁、在什么场景、解决什么问题、受什么约束。
- 第一模块是否让最着急的学员拿到独立成果。
- 每个事实、数字、案例和原话能否回到材料来源。
- 逐字稿是否像老师连续讲课，段落是否承载完整教学动作。
- 审校是否指出具体位置与证据，而非给“AI 含量”分数。
- 中断后是否从真实产物恢复，而非重写已完成章节。
- 飞书写入是否经过正文回读与文件夹归属验证。
- 单次满意是否只记录为作品信号，没有直接污染长期规则。
